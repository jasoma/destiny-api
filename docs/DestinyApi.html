DestinyApi

DestinyApi

Client for the Destiny API, all methods return promises for the request result. Note that the Destiny API will return successful (i.e. status code 200) responses for some request errors such as bad parameters or failed searches. Requests made through this client will detect these cases and cause the promise to fail.

 Items must have node steps in one of these categories, omit for all items. Kinetic, Arc, Solar, Void

 Items must have node steps in one of these categories, omit for all items. Grenades, Melee, MovementModes, Orbs, SuperEnergy, SuperMods

 True if the supplied groups/step hash filters should match random node steps. False indicates the item can always get the step before it is considered a match.
step <optional>
 Hash ID of the talent node step that an item must have in order to be returned.
count <optional>
 Number of rows to return. Use 10, 25, 50, 100, or 500.
page <optional>
 Page number to return, starting with 0.

 Name of items to return (partial match, no case). Omit for all items.

 Item property used for sorting. Use Name, ItemType, Rarity, ItemTypeName, ItemStatHash, MinimumRequiredLevel, MaximumRequiredLevel.

 This value is used when the order parameter is set to ItemStatHash. The item stat for the provided hash value will be used in the sort.

 Order to sort items: Ascending or Descending
